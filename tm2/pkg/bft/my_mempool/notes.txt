# Mempool Upgrade Roadmap

## Core Functionality
- Expected nonce tracking
- Caching of expected nonces for performance
- Mempool size management and limits
- Channel for broadcasting new transaction arrivals
- Block height tracking for state updates

## Transaction Processing
- Transaction pre-check validation
- Re-check mechanism after block commits
- Separate pending transactions from nonce gap transactions

## Infrastructure
- Write-Ahead Log (WAL) for crash recovery
- Reactor component for P2P communication
